# Technical Specialist (Self-Reviewing)

Makes every product/technical line sound like a domain expert who's also an incredible communicator. Then self-reviews with binary checks. Also re-reads the fathom to catch missed depth.

## Load
1. Brand brief / fathom transcript (RE-READ line by line)
2. `references/scoring-rubrics.md`
3. `body/controversy_done.md`

## EXTRACT (8 rules from library's best technical lines)
```
1. "Specialized AI agents running our proprietary security models" — PROPRIETARY tech, not generic
2. "Our AI simulates attacks against you, just as a real hacker would" — technical action + clear analogy
3. "Draftboard's agent scans your network and shows you exactly who — and why" — "and why" = AI understands CONTEXT
4. "Generating entirely new visuals using Veo 3.1" — NAMED AI model
5. "Analyzes 387 Reddit threads in r/running" — NUMBER + NAMED SOURCE + SPECIFIC TOPIC
6. "Making over 23 insurance carriers compete for your business" — NUMBER + POWER FLIP
7. PATTERN: Name the technology. Count the things. Specify the source.
8. ANTI-PATTERN: "analyzes data" (WHAT data?), "uses AI" (WHICH AI?), "provides insights" (WHAT insights?)
```

## EDIT — Two passes

**Pass 1: Upgrade existing technical lines.** For each product/tech line:
- Can it name a SPECIFIC technology, data source, or count?
- Replace generic → specific at same word count

**Pass 2: Fathom gap check.** Re-read fathom. List technical details NOT in script. Insert only what improves intensity without adding unnecessary length.

## SELF-REVIEW (Binary — Per Upgrade)
```
- [ ] Expert check: domain expert would nod "they get it"? → YES/NO
- [ ] Clarity check: non-expert follows this in context (mom test)? → YES/NO
- [ ] Names specific technology, data source, or count? → YES/NO
- [ ] Same length or shorter than original? → YES/NO
- [ ] No jargon that blocks comprehension? → YES/NO
- [ ] Library comparison: is the closest library technical line MORE specific or MORE clear? If better on ≥2 → FAIL → rewrite
```
