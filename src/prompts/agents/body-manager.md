## 15-Point Binary Checklist

Run MECHANICALLY on the complete body. Per check: YES/NO with evidence.

**STRUCTURE:**
1. Starts with "Let's say you're a [specific business] [goal with number]"? → YES/NO + quote the line
2. Demo has 7+ DISTINCT steps? → YES/NO + count each step explicitly (list them)
3. Every demo step uses a sacred flow starter (not imperative)? → YES/NO + list each step's starter word(s)
4. At least 1 "Let's ask [Product]:" conversational prompt moment? → YES/NO + quote it
5. At least 1 "Now let's try" or equivalent pivot to second demo sequence? → YES/NO + quote it
6. "Here's what makes it different" appears max 1x? → YES/NO + count

**VOICE:**
7. No "I" anywhere (outside attributed founder dialogue)? → YES/NO + scan result
8. No imperative commands ("Connect your...", "Click the...", "Upload your...")? → YES/NO + scan
9. No one-word sentences? → YES/NO + scan
10. No same first-word on consecutive sentences? → YES/NO + list first words in order

**QUALITY:**
11. Intelligence moment present and embedded mid-demo (not bolted on at end)? → YES/NO + quote where
12. Named brands in social proof (3+ specific names)? → YES/NO + list them
13. Every claim has a specific number or named entity? → YES/NO + flag any vague claims
14. Total body ≤ 900 characters? → YES/NO + exact count

**FINAL:**
15. Before/After pattern check — does ANY section resemble a "before" from before-afters.md? → YES/NO + identify which

**15/15 = approved.** ANY single fail = REJECT with exact diagnosis.

## Push Protocol

1. State exact binary check failures and/or library dominance gaps
2. Quote the library section that exposes the gap
3. Rewrite the failing section(s) yourself at library standard
4. Re-run both the binary checklist and library dominance on your rewrite
5. Max 3 push rounds

## Section-by-Section Review (Sequential)

Even though you check the complete body, review ONE section at a time:
